=========================================================
UAV SIMULATION ENGINE - FUTURE OPTIMIZATIONS

This document outlines the roadmap for scaling the simulation
to massive grid sizes (e.g., 10,000 x 10,000) and real-time
swarm processing.

SPATIAL PARTITIONING (Algorithmic)

Technique: Quadtrees or Spatial Hashing.

Why: Currently, the engine checks every single threat for
every single grid cell (O(N * M)). By dividing the map into
sectors, a cell only checks threats physically inside its
sector, reducing operation time to O(N log M).

GPU COMPUTE SHADERS (Hardware)

Technique: CUDA / OpenCL / Vulkan Compute.

Why: Generating a grid of independent probability numbers
is an "embarrassingly parallel" problem. Moving the
calculateChunk nested loops to a Graphics Card would calculate
millions of cells simultaneously, offering a 100x to 1000x speedup.

SIMD VECTORIZATION (Hardware)

Technique: AVX2 / AVX-512 Intrinsics.

Why: Instead of doing math on one cell at a time, modern
CPUs can pack 8 cells into a 256-bit register and calculate
them all in a single clock cycle.

ADVANCED PATHFINDING (A* Upgrades)

Theta* (Any-Angle A*): Standard A* produces zig-zag paths
(limited to 8 directions). Theta* allows line-of-sight
movement, creating smooth, realistic flight trajectories.

Jump Point Search (JPS): Skips evaluating 0% risk areas
("jumping" over empty skies), drastically reducing memory
overhead during the path search.

Brushfire/Distance Transform: Adds a repulsion gradient
around NFZs, pushing the A* path to the dead-center of safe
corridors instead of hugging the edge of a threat zone.
